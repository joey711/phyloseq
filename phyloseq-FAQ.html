I tried reading my biom file using phyloseq, but it didnt work. Whats wrong?

biom-format Versions 1 and 2

HDF5 (Version 2.0) biom-format has beta support

It is currently supported in the development version of phyloseq, via the new beta/development package called biomformat on BioC and GitHub:

https://github.com/joey711/biomformat

Not every data component is included in .biom files

Even though the biom-format supports the self-annotated inclusion of major components like that taxonomy table and sample data table, many tools that generate biom-format files (like QIIME, MG-RAST, mothur, etc.) do not export this data, even if it is provided as an input file. Furthermore, even though I have proposed it to the biom-format team, there is currently know support (or timeline for support) for inclusion of a phylogenetic tree within a .biom file.

A number of tutorials are available demonstrating how one can add components to a phyloseq object after it has been created/imported. The following tutorial is especially relevant http://joey711.github.io/phyloseq-demo/import-biom-sd-example.html

psmelt and ggplot2

Originally an internal phyloseq tool used by several phyloseq plot_ functions, the psmelt function converts your phyloseq object into a data.frame that is very friendly for defining a custom ggplot2 graphic. There are lots of great documentation examples for ggplot2 in general, and phyloseq in particular.

Submit a Pull Request (Advanced)

If your new custom plot function is awesome and you think others might use it, add it to the plot-methods.R function and submit a pull request on GitHub. Please include example and test code. Ill try and add it to the package by the next release. I will also give you authorship cred in the function doc.
